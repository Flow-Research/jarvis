            What this proves

            Jarvis is not the chat UI and not the runtime itself. Jarvis is the collaboration layer that lets an agent work autonomously inside policy, ask when it hits a boundary, learn from review, and carry evidence forward.

            Human

            Provides intent, judgment, review, corrections, and approval.

            Agent

            Plans, executes, uses tools, collects evidence, and learns.

            Jarvis

            Owns WorkSession, memory, policy, requests, skills, and evidence.

            Runtime

            Provides durable execution, files, sandbox, streaming, and recovery.
